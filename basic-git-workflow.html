1
Checkout the Project
Clone or checkout files from the repository to your working directory. This pulls a specific version of the project for you to work on.
git checkout / git clone
.GIT REPOSITORY WORKING DIR
2
Stage Fixes
After modifying files in your working directory, stage them to mark them for inclusion in the next commit. This adds a snapshot to the staging area.
git add <files>
WORKING DIR STAGING AREA
3
Commit
Take the files from the staging area and store a permanent snapshot in the .git directory (repository). This creates a new commit in the project history.
git commit -m "message"
STAGING AREA .GIT REPOSITORY
// THE THREE STATES
  • Modified: Changed files in working directory, not yet staged
  • Staged: Marked modified files for next commit snapshot
  • Committed: Data safely stored in local .git database
