1
Dictator Pushes to Blessed
The dictator (project maintainer) pushes to their public blessed repository.
DICTATOR
2
Contributors Clone & Fork
Contributors clone the repository, fork their topic branch, and make changes locally.
DEVELOPERS
3
Contributors Push Topic Branch
Contributors push their topic branch to their own public repository.
DEVELOPERS
4
Pull Request to Lieutenant
Contributors send lieutenants an email/PR asking them to pull changes.
DEVELOPERS LIEUTENANTS
5
Lieutenants Merge Locally
Lieutenants add contributor's repo as remote and merge to their master branch.
LIEUTENANTS
6
Lieutenants Request Dictator Pull
When sufficient changes accumulate, lieutenants send the dictator a pull request.
LIEUTENANTS DICTATOR
7
Dictator Merges Lieutenant Changes
Dictator adds lieutenants' repos as remotes and merges their master branches locally.
DICTATOR
8
Dictator Pushes to Blessed
Dictator pushes merged changes to the blessed repository. Developers can now rebase.
DICTATOR
// KEY CHARACTERISTICS
  • Used by projects with hundreds of collaborators
  • Lieutenants oversee specific parts of the repository
  • Dictator delegates work and collects large subsets
  • Famous example: Linux Kernel development
  • Developers rebase their work on top of master
